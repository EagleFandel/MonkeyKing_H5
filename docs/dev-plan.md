# 开发计划（MVP 先行）

## 时间轴（单前端）

## M0：基础搭建（第 1 周）
- 初始化 Vue3+Vite+TS 工程。
- 搭建路由、状态管理、全局样式 token。
- 建立 MSW Mock 与 API 适配层。
- 输出业务总览、需求目录、PRD、Design 初稿。

## M1：四场景主链路（第 2~3 周）
- 完成四场景的需求表单页与智能体补全。
- 完成匹配列表与详情页。
- 完成订单创建与支付页。
- 初步可演示端到端流程。

## M2：通用能力补齐（第 4~5 周）
- 实名认证页、消息中心、投诉页、个人中心。
- 返利汇总与提现模拟。
- 订单历史和消息红点同步。

## M3：稳定性与验收（第 6~7 周）
- 完善异常态与空态。
- 补齐单元测试、E2E 用例。
- 构建发布文档与交付说明。

## 任务拆解
- 架构组：路由、目录、API、Mock、Store。
- 页面组：四场景 + 通用页面。
- 组件组：表单、候选卡、智能体、支付面板、头部统计。
- 文档组：PRD、Design、API、测试、计划。
- 质量组：Vitest、Playwright、构建验证。

## 风险与缓冲
- 风险：场景跨度大、需求变更频繁。
  - 对策：冻结核心字段，扩展项通过配置化补充。
- 风险：智能体规则覆盖不足。
  - 对策：优先稳定抽取预算/时间/描述，逐步增加规则。
- 风险：单人并行压力。
  - 对策：先交付主链路，再做体验增量。
