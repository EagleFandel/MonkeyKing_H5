# 悟空哆哆 H5 业务总览

## 1. 项目定位
- 产品定位：一个“发现大学生”的 H5 平台，连接需求方与大学生供给方。
- 核心价值：通过智能体结构化需求、智能匹配、支付与投诉闭环，提升撮合效率与合作安全性。
- 目标：四大场景同一期上线并可端到端演示。

## 2. 角色与价值主张
- 创业者：快速找到匹配开发者团队，缩短项目启动周期。
- 家长：精准筛选大学生家教，付费后获取联系方式。
- 陪练需求用户：按时间地点水平匹配网球陪练并预约支付。
- 大学生：生成人才画像，持续获得精准兼职推荐并积累成长记录。

## 3. 四大核心场景
### 3.1 创业者找开发者
1. 提交项目需求（类型/技术栈/预算/周期/性格偏好）
2. 智能体抽取与补问，生成项目报告
3. 匹配候选开发者并查看详情
4. 选择开发者并创建订单
5. 支付后进入协作阶段（本期以流程演示为主）

### 3.2 家长找家教
1. 填写年级、科目、时段、地点与预算
2. 智能匹配家教候选
3. 查看候选详情
4. 支付小额费用获取联系方式

### 3.3 网球陪练
1. 填写运动时间地点、水平和提升目标
2. 匹配陪练候选
3. 沟通确认后发起预约
4. 支付完成预定

### 3.4 大学生找兼职
1. 填写技能、熟练度、可工作时间和薪资预期
2. 智能体辅助生成人才画像
3. 推荐匹配兼职机会
4. 一键申请并等待反馈

## 4. 通用能力
- 实名认证：核心流程前置能力。
- 消息中心：系统、互动、返利消息统一沉淀。
- 个人中心：身份状态、记录聚合、返利提现。
- 投诉机制：场景化投诉入口与状态跟踪。
- 模拟支付：微信/支付宝双通道状态机。

## 5. 成功指标（MVP）
- 四场景主链路可跑通。
- 关键页面统一视觉和交互规则。
- Mock API 与前端契约一致，便于后续替换真实后端。
- 单元测试 + E2E 核心用例可执行。
