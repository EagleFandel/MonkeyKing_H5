# 悟空哆哆 H5 Design V2 · 文案语气规范

## 1. 语气目标
- 去除“机械提示感”和“系统公告感”。
- 强化“陪伴式引导 + 明确行动 + 清晰结果”。

## 2. 文案公式
- 说明型：`当前状态 + 你将获得什么`。
- 引导型：`下一步动作 + 预期时间/结果`。
- 错误型：`问题原因 + 立即修复动作`。
- 成功型：`结果确认 + 推荐下一步`。

## 3. 示例（推荐替换）
- 原：`提交失败`
- 新：`提交未成功，请检查必填项后重试`

- 原：`暂无数据`
- 新：`暂时还没有匹配结果，试试放宽预算或时间范围`

- 原：`支付成功`
- 新：`支付已完成，合作方已收到通知，建议现在发起首次沟通`

## 4. 禁止项
- 禁止连续感叹号、全大写、负面指责式文案。
- 禁止缺少动作指令的“死提示”。

## 5. 微文案规则
- 按钮文案优先动词开头：`去匹配`、`立即预约`、`确认支付`。
- Toast 不超过 18 字，且必须可理解。
- 输入 placeholder 避免泛化，提供真实示例。
