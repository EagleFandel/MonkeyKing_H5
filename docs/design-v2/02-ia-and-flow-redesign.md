# 悟空哆哆 H5 Design V2 · 信息架构与流程重排

## 1. 全局 IA（重构后）
- `首页 Home`：品牌叙事、场景分流、智能体引导、信任背书、近期动态。
- `场景页 Scene`：统一五段式结构（目标说明 → 结构化输入 → 智能补全 → 推荐结果 → 行动确认）。
- `通用页 Common`：支付、消息、个人中心、认证、投诉、候选详情。

## 2. 导航策略
- 顶部：品牌 Logo、消息入口、动态背景（随路由 meta 切换 tone）。
- 底部主导航：`首页`、`发现兼职`、`找项目`、`找家教`、`我的`。
- 陪练入口：位于首页场景入口和可检索入口，不单独占据底部 tab，降低底栏拥挤。

## 3. 首页结构重排（Hero 叙事）
1. Hero：一句价值主张 + 主 CTA（立即开始）+ 次 CTA（看看案例）。
2. 场景入口：4 个入口卡片（创业/家教/陪练/兼职），提供“时间预估 + 成功率标签”。
3. 智能体入口：可展开浮层，支持直接输入自然语言需求。
4. 信任体系：实名认证、支付托管、投诉处理、推荐返利。
5. 动态反馈：最近成功匹配、平均响应时间、满意度。

## 4. 场景页统一结构（五段式）
1. **目标**：本场景价值说明 + 成功标准。
2. **输入**：结构化表单（可分组折叠）。
3. **智能补全**：哆哆抽取 + 追问 + 回填。
4. **推荐**：候选列表/职位列表 + 筛选条 + 推荐理由。
5. **行动**：支付/预约/申请/沟通。

## 5. 四场景关键流程（重排版）

### 5.1 创业者找开发者
- 填写需求 → 智能补全 → 生成报告 → 推荐团队 → 支付下单 → 协作启动。
- 将“报告”和“匹配”并列入口，避免强制单一顺序。

### 5.2 家长找家教
- 填写需求 → 智能补全 → 匹配家教 → 查看详情 → 付费解锁联系方式。
- 在支付前增加“联系说明卡”，明确费用用途和退款条件。

### 5.3 网球陪练
- 填写训练目标 → 智能补全 → 匹配陪练 → 沟通确认 → 预约支付 → 评价反馈。
- 支持“基础陪练/技巧指导”模式切换。

### 5.4 大学生找兼职
- 填写技能 → 智能补全 → 生成人才画像 → 推荐岗位 → 申请反馈。
- 在申请前展示“匹配差距提示”，提示需补充信息项。

## 6. 关键路径缩短策略
- 核心流程控制在 4 步以内（不含详情页）。
- 关键输入项先展示必填，选填折叠。
- 所有场景主 CTA 固定底部可见（移动端 thumb-zone 优化）。

## 7. 异常路径
- 未认证访问核心功能：统一拦截 + 说明收益 + 一键去认证。
- 匹配为空：提供“调整条件建议”+ “一键放宽筛选”。
- 支付失败：保留订单 + 提供重试和更换渠道。
- 提交失败：保留已填数据 + 错误原因 + 重新提交。
