# 悟空哆哆 H5 Design V2 · 设计原则

## 1. 设计目标
- 从“能用”升级为“有品牌辨识、有沉浸感、有行动引导力”的移动端体验。
- 保留橙蓝品牌资产，扩展为可系统化实现的视觉层级（色阶、材质、动效、状态）。
- 建立可复用的交互与组件规范，避免页面各自为战、风格割裂。

## 2. 五大原则
1. **品牌叙事优先**：首屏必须先回答“我是谁、我能帮你做什么、下一步做什么”。
2. **强行动导向**：每页提供 1 个主 CTA + 1 个次 CTA，避免决策瘫痪。
3. **状态可感知**：默认/加载/空态/错误/成功 5 态完整定义且视觉区分明显。
4. **交互连续性**：点击反馈、卡片展开、页面转场、支付反馈遵循统一节奏。
5. **工程可落地**：所有规则可映射到 token、组件 props、路由 meta、状态机。

## 3. 体验关键词
- 年轻活力：有能量，但不花哨。
- 专业可信：层级清晰，关键数据可读。
- 沉浸顺滑：高动效但不眩晕，过渡自然。
- 全程可控：任何关键动作都有反馈、有回退、有恢复路径。

## 4. 非目标（本轮不做）
- 不引入重型 3D 引擎与高成本资产管线。
- 不变更后端业务协议与核心交易流程。
- 不使用版权不明素材包，优先代码绘制与轻量 SVG。

## 5. 验收红线
- 关键页面 5 态覆盖率必须 100%。
- 全站主流程中不得出现“点击无反馈 > 300ms”的无感知交互。
- 同级组件在不同页面的尺寸/间距/圆角偏差不得超过 1 个 token 档位。
