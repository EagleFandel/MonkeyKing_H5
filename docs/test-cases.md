# 测试用例清单

## 1. 单元测试

### 1.1 智能体规则引擎
- 输入包含“项目/预算”关键词，能识别为 `project`。
- 无预算时返回追问问题。
- 预算数字可被正确抽取。

### 1.2 匹配评分
- 权重加权结果符合预期。
- 同分与不同分排序稳定。

### 1.3 支付状态机
- `pending -> paid` 合法。
- `pending -> failed` 合法。
- `paid -> pending` 非法并抛错。

## 2. 集成测试
- 首页进入任意场景可成功跳转。
- 创建订单后在支付页可见当前订单。
- 消息中心“全部已读”后未读数归零。

## 3. E2E 必测场景
- 创业者：填写需求 -> 匹配 -> 创建订单 -> 支付成功。
- 家长：填写需求 -> 匹配 -> 创建联系方式订单 -> 支付成功。
- 陪练：填写需求 -> 匹配 -> 预约订单 -> 支付成功。
- 兼职：生成画像 -> 推荐 -> 申请成功。
- 通用：认证提交、投诉提交、返利提现。

## 4. 验收口径
- 所有核心页面可打开且无阻塞报错。
- Mock 接口成功返回率 100%。
- `npm run build` 与 `npm run test:unit` 通过。
