# 悟空哆哆 H5 Design System

## 1. 设计目标
- 构建统一、可信、年轻化的移动端体验。
- 保证四大场景视觉一致、交互一致、组件可复用。
- 在 375~480px 屏宽下保持稳定可读与易点击。

## 2. 视觉 Token

### 2.1 颜色
- 主品牌色：`#FF7A00`（按钮、关键数值、主 CTA）
- 辅助色：`#5BA8FF`（智能体、次级 CTA、信息强调）
- 页面背景：`#F7FAFF`
- 卡片背景：`#FFFFFF`
- 主文字：`#1F2937`
- 次文字：`#6B7280`
- 成功：`#16A34A`
- 警告：`#F59E0B`
- 危险：`#EF4444`

### 2.2 圆角
- 卡片：`10px`
- 输入框：`8px`
- 按钮：`12px`

### 2.3 阴影
- 卡片阴影：`0 8px 18px rgba(0,0,0,0.08)`
- 顶部/底部导航：`0 2px 12px rgba(0,0,0,0.08)`

### 2.4 间距系统
- 基础：`4/8/12/16/20/24`
- 页面主间距：`12px`
- 组件内边距：`10~14px`

## 3. 字体与排版
- 字体：微软雅黑 / 思源黑体 / PingFang SC。
- 层级：
  - H1：20px / 700
  - H2：18px / 700
  - H3：16px / 700
  - Body：14px / 400
  - Caption：12px / 400
- 行高：1.5~1.8。
- 关键值（匹配度、金额、时薪）统一加粗并用主品牌色。

## 4. 核心组件规范

### 4.1 导航
- 顶部固定导航：Logo + 消息入口 + 未读红点。
- 底部固定导航：5 项入口，当前项高亮。

### 4.2 表单
- 单输入最小点击区 `44x44`。
- 标签在上，输入在下。
- 必填项在文案和交互中明确提示。

### 4.3 卡片
- 候选卡片统一结构：标题、标签、经验、评分、价格、动作按钮。
- 支持跨场景复用（开发者/家教/陪练/兼职）。

### 4.4 智能体面板
- 头部：头像 + 说明
- 内容：文本输入 + “智能提取”按钮
- 输出：意图、报告草稿、追问

### 4.5 支付面板
- 订单金额、支付渠道单选、确认/取消按钮。
- 状态颜色统一：pending=warning、paid=success、failed=danger。

## 5. 交互规范
- 按钮反馈：120ms 颜色变化。
- 路由切换：不超过 1 秒，必要时 toast 提示。
- 表单链路：允许“手填 + 智能体补全”双通道。
- 异常处理：API 失败统一 toast 错误提示。

## 6. 可用性与可访问性
- 触控目标最小 44x44。
- 文字对比符合移动端可读性（主文字优先深色）。
- 状态表达“颜色 + 文案”双通道，避免仅靠颜色区分。
