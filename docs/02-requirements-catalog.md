# 需求目录（MoSCoW）

## Must Have（本期必须）
- 四大场景完整主链路
  - 创业者：需求填写、报告生成、候选匹配、支付下单
  - 家长：需求填写、家教匹配、付费获取联系方式
  - 陪练：需求填写、陪练匹配、预定支付
  - 兼职：技能填写、画像生成、兼职推荐、申请提交
- 全局能力
  - 实名认证页面与校验流程
  - 消息中心（读未读）
  - 个人中心（返利统计、提现模拟）
  - 投诉入口与提交
  - 支付状态机（pending/paid/failed）
- 平台基础架构
  - Vue3+Vite+TS
  - Router + Pinia
  - MSW Mock API + 契约统一响应结构
  - 单元测试与 E2E 测试框架可运行

## Should Have（本期建议）
- 智能体规则引擎补问策略
- 候选卡片统一组件并支持跨场景复用
- 返利台账列表与提现后余额动态变化
- 候选详情页与高频入口跳转

## Could Have（可选增强）
- 报告导出为真实 PDF
- 智能体接入真实大模型
- 多维筛选器（评分/距离/经验）
- 画像动态成长曲线可视化

## Won't Have（本期不做）
- 真实支付网关、实名认证 OCR/人脸核验
- 生产级后端数据库与风控策略
- 多终端原生推送能力
