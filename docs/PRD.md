# 悟空哆哆 H5 PRD

## 1. 产品概述
- 产品名：悟空哆哆 H5
- 目标：打造“发现大学生”的移动端撮合平台，覆盖四大业务场景。
- 发布形态：前端 H5 + Mock API 演示版。

## 2. 业务目标与 KPI
- 匹配效率：需求提交到首批候选曝光时长 < 10 秒（Mock环境）。
- 支付转化：完成下单后支付成功率 > 85%（模拟链路统计）。
- 体验指标：关键路径完成率 > 70%。

## 3. 用户角色与权限
- 创业者：可发起项目需求、选择开发者并支付。
- 家长：可发起家教需求、付费获取联系方式。
- 学生：可发布技能画像并申请兼职。
- 陪练需求用户：可发起陪练需求并支付预约。
- 全角色：可查看消息、提交投诉、管理个人中心。

## 4. 全局业务流程
1. 首页选择场景或进入智能体。
2. 场景化表单填写 + 智能体辅助抽取。
3. 平台生成报告或画像并返回候选列表。
4. 用户查看详情并发起下单/申请。
5. 支付状态更新，消息中心通知结果。
6. 个人中心沉淀记录与返利。

## 5. 功能规格

### 5.1 首页
- 模块：欢迎语、四场景快捷入口、核心优势、底部条款。
- 交互：点击快捷卡片进入对应场景。

### 5.2 创业者找开发者
- 输入字段：项目类型、技术栈、预算区间、周期、性格偏好、描述。
- 输出：项目报告 + 开发者候选列表。
- 关键动作：选择候选 -> 创建订单 -> 跳转支付。

### 5.3 家长找家教
- 输入字段：年级、科目、时段、线上/线下、地点、预算、个性化需求。
- 输出：家教候选列表。
- 关键动作：支付联系方式订单。

### 5.4 网球陪练
- 输入字段：运动类型、时间段、地点、水平、预算、提升目标。
- 输出：陪练候选列表。
- 关键动作：创建预约订单并支付。

### 5.5 大学生兼职
- 输入字段：技能、熟练度、可工作时间、薪资预期、兴趣、介绍。
- 输出：人才画像 + 兼职推荐列表。
- 关键动作：提交兼职申请。

### 5.6 实名认证
- 页面字段：角色、姓名、证件号、人脸 token（mock）。
- 规则：通过前禁止使用关键交易能力（在演示版中通过页面引导体现）。

### 5.7 支付中心
- 渠道：微信、支付宝。
- 状态机：`pending -> paid` / `pending -> failed`。
- 结果：订单状态更新并显示历史记录。

### 5.8 消息中心
- 类型：system / interactive / referral。
- 支持：刷新、全部标已读、未读计数。

### 5.9 投诉反馈
- 输入：场景、对象ID、原因、证据。
- 输出：投诉编号与处理提示。

### 5.10 个人中心
- 展示：实名状态、返利汇总、返利明细。
- 动作：提现模拟、跳转消息/投诉/支付。

## 6. 接口契约
- 响应统一：`{ code, message, data, traceId }`
- 详见 `docs/api-contract.md`

## 7. 非功能需求
- 适配：375~480px 移动端优先。
- 点击最小区域：44x44。
- 首屏交互反馈：按钮/跳转有明确视觉反馈。

## 8. 验收标准
- 四场景主流程可独立完成。
- 关键模块（认证、支付、投诉、消息、返利）可演示。
- 文档与代码一致，接口联调路径清晰。
